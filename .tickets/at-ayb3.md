# Add invite-via-SMS admin tool to packages/backend

Port the admin invite tool from [friends-and-family-supabase/backend/admin.py](https://github.com/cdiddy77/friends-and-family-supabase/blob/main/backend/admin.py) into `packages/backend` as a Click CLI command. This tool creates a Supabase user (if not exists), inserts an invite row in the `invites` table, and optionally sends the activation link via Twilio SMS.

The frontend activation flow (`/activate?code=...`) and session persistence already work ‚Äî this ticket only adds the backend admin tooling to create and send invitations.

## Details

### What the tool does (from friends-and-family-supabase)

1. **`create_user_if_not_exists(phone)`** ‚Äî Uses Supabase Admin API to create a user with the given phone number (using a `{phone}@example.com` dummy email convention). If user already exists, finds them by phone via `admin.list_users()`.
2. **`create_invite(phone)`** ‚Äî Creates an invite row in the `invites` table with `user_id` and a 2-week `expires_at`. The `code` column is auto-generated by the database.
3. **`send_sms(phone, code, base_url)`** ‚Äî Uses Twilio to send an SMS with the activation link (`{base_url}/activate?code={code}`).

### Where to put it

- Create `packages/backend/src/boards/tools/` directory
- Add `packages/backend/src/boards/tools/__init__.py`
- Add `packages/backend/src/boards/tools/invite.py` ‚Äî core logic (create user, create invite, send SMS)
- Add `packages/backend/src/boards/tools/cli.py` ‚Äî Click CLI entry point
- Register `boards-invite = "boards.tools.cli:main"` in `pyproject.toml` `[project.scripts]`

### CLI interface

```bash
# Create invite and print the link (no SMS)
boards-invite create +15555555555 --url https://myapp.example.com

# Create invite and send via Twilio SMS
boards-invite create +15555555555 --url https://myapp.example.com --send
```

### Dependencies to add

- `twilio>=9.9.0` ‚Äî add as a new optional dependency group `[project.optional-dependencies]` under `sms-twilio` (not a core dep since not all deployments need SMS)
- `supabase>=2.0.0` ‚Äî already available as optional dep (`auth-supabase` or `storage-supabase`)
- `click>=8.0.0` ‚Äî already a core dependency
- `python-dotenv>=1.0.0` ‚Äî already a core dependency

### Environment variables

The tool reads from `.env` or environment:
- `SUPABASE_URL` (required)
- `SUPABASE_SERVICE_ROLE_KEY` (required ‚Äî must be service role for admin API)
- `TWILIO_ACCOUNT_SID` (required only with `--send`)
- `TWILIO_AUTH_TOKEN` (required only with `--send`)
- `TWILIO_PHONE_NUMBER` (required only with `--send`)

### Existing frontend pieces (already implemented, no changes needed)

- `/activate?code=...` page redeems invite codes via `/api/activate` route
- `/api/activate` validates the code, marks it redeemed, and issues a JWT
- `SupabaseProvider` picks up the session and authenticates the user
- Supabase middleware persists sessions across page loads via cookies
